# 写作规范

这份文档记录目前已经明确下来的写作要求，供后续持续复用。

## 当前适用场景

- AI 思考
- AI 使用技巧
- AI 工作流
- 人与 agent 的协作关系
- 适合写成公众号短篇观点文的话题

## 当前默认写法

### 1. 先给判断

- 开头尽快亮观点
- 不展示长讨论过程
- 不绕弯子，不先铺垫一大段背景
- 如果有足够强的一句话摘要，可以直接以前置 `TL;DR` 形式放在开头

### 2. 观点要硬，语言要顺

- 观点要鲜明
- 语言可以口语化
- 不能只是口水话，要保留解释力

### 3. 优先共鸣，而不是完整

- 让读者感觉“这就是我最近的感受”
- 多写时代变化对普通人的影响
- 少写过于抽象的宏大叙事

### 4. 结构要短

- 优先 3 到 4 个自然段或小段落
- 不要把一篇 500 字短文拆成太多章节
- 每一段都要服务中心判断

### 5. 要有可传播句

- 文章里尽量有一两句能单独成立的话
- 这类句子应该适合被截图、引用、转述
- 这些句子可以适度加粗，方便公众号读者扫读和记忆

### 6. 成稿要可直接发布

- 放进 `articles/` 的内容，默认就是最终发布版
- 不保留候选标题、摘要说明、注释、编辑提示
- 目标是让文章可以一键复制，而不是还要二次清理
- 如果目标平台不支持本地图片，应额外生成平台可直接粘贴的发布版
- `articles/` 目录只放文章文件，不放图片资源

### 7. 配图要一起考虑

- 公众号文章默认不是只有文字
- 如果可以直接生成图，就直接生成并插入到文章合适位置
- 不要只给配图建议，把执行工作留给你
- 配图优先承担两件事：
  - 强化核心判断
  - 打断阅读疲劳

### 8. mdnice 发布版

- `mdnice` 不应依赖本地相对路径图片
- 默认使用 GitHub 仓库公开图片 + jsDelivr CDN
- 需要单独生成一份 `-mdnice.md` 文件，放在 `publishing/` 下，供直接粘贴到 `editor.mdnice.com`

## 当前明确不做的事

- 不把文章写成讨论记录
- 不在文中强调“这不是学术分类”
- 不为了显得严谨而削弱观点锋芒
- 不把短文写成过度完整的大长文
- 不把最关键的一句话藏到文末

## 当前标题偏好

- 一句话直接亮结论
- 尽量让读者一眼看懂文章主张
- 个人思考型内容要避免标题卸力
- 少用“可能”“也许”“大概”这类模糊词
- 不必写成标题党，但要明确站位
- 已经迫在眼前的变化，不要写成“未来某天会发生”
- 如果不加时间词也能成立，就不要额外加“现在”

## 当前目标

这类文章的目标不是面面俱到，而是：

- 让读者快速读懂一个判断
- 对一部分人形成共鸣
- 让文章有一定转发和讨论价值
