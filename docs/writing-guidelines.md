# 写作规范

这份文档记录目前已经明确下来的写作要求，供后续持续复用。

## 当前适用场景

- AI 思考
- AI 使用技巧
- AI 工作流
- 人与 agent 的协作关系
- 适合写成公众号短篇观点文的话题

## 当前默认写法

### 1. 先给判断

- 开头尽快亮观点
- 不展示长讨论过程
- 不绕弯子，不先铺垫一大段背景
- 如果有足够强的一句话摘要，可以直接以前置 `TL;DR` 形式放在开头
- `TL;DR` 默认保留一个空的引用行，增强视觉停顿

### 2. 观点要硬，语言要顺

- 观点要鲜明
- 语言可以口语化
- 不能只是口水话，要保留解释力

### 3. 优先共鸣，而不是完整

- 让读者感觉“这就是我最近的感受”
- 多写时代变化对普通人的影响
- 少写过于抽象的宏大叙事

### 4. 结构要短

- 优先 3 到 4 个自然段或小段落
- 不要把一篇 500 字短文拆成太多章节
- 每一段都要服务中心判断

### 5. 要有可传播句

- 文章里尽量有一两句能单独成立的话
- 这类句子应该适合被截图、引用、转述
- 这些句子可以适度加粗，方便公众号读者扫读和记忆

### 6. 成稿要可直接发布

- 放进 `articles/` 的内容，默认就是最终发布版
- 不保留候选标题、摘要说明、注释、编辑提示
- 目标是让文章可以一键复制，而不是还要二次清理
- `articles/` 目录只放文章文件，不放图片资源
- `articles/` 中的文章保留对 `assets/` 的本地引用

### 7. 配图要一起考虑

- 公众号文章默认不是只有文字
- 如果可以直接生成图，就直接生成并插入到文章合适位置
- 不要只给配图建议，把执行工作留给你
- 配图优先承担两件事：
  - 强化核心判断
  - 打断阅读疲劳
- 配图不是为了把文章再复述一遍，而是为了让观点更容易被分享
- 一张图最好只表达一句话
- 图中文字不要密，不要杂，不要像汇报材料
- 如果读者看完没有截图转发的欲望，说明图做错了
- 水印必须避开边框、线条和主要内容，默认预留独立安全区
- 如果图片右下角已有视觉元素，先改版式，再放水印，不允许水印与边框贴在一起
- 默认补一张公众号横版封面，比例为 `2.35:1`
- 公众号横版封面与正文顶部封面不是一回事，需单独考虑
- 公众号横版封面优先命名为 `wechat-cover.jpg`

### 8. 图片链接策略

- 不再使用 GitHub CDN 作为公众号图片源
- `mdnice` 采用手动上传图床的方式
- 文章图片保留在 `assets/`
- 同时额外复制一份到本地 `~/Downloads`，按 `1`、`2`、`3` 命名，方便手动上传
- 图片水印默认使用 `@Bill的精神时光屋`

## 当前明确不做的事

- 不把文章写成讨论记录
- 不在文中强调“这不是学术分类”
- 不为了显得严谨而削弱观点锋芒
- 不把短文写成过度完整的大长文
- 不把最关键的一句话藏到文末
- 不把已经稳定形成的格式习惯只留在单篇文章里

## 提交约定

- 写作规范、流程、格式记忆等基础信息的修改，单独提交一个 commit
- 这类 commit message 直接写本次修改的主要内容
- 每天文章相关改动单独提交一个 commit
- 文章 commit message 直接使用当天文章标题
- 不把“规范修改”和“当天文章”混在同一个 commit 里

## 当前标题偏好

- 一句话直接亮结论
- 尽量让读者一眼看懂文章主张
- 个人思考型内容要避免标题卸力
- 少用“可能”“也许”“大概”这类模糊词
- 不必写成标题党，但要明确站位
- 已经迫在眼前的变化，不要写成“未来某天会发生”
- 如果不加时间词也能成立，就不要额外加“现在”

## 当前目标

这类文章的目标不是面面俱到，而是：

- 让读者快速读懂一个判断
- 对一部分人形成共鸣
- 让文章有一定转发和讨论价值
